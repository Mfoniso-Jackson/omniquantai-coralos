# Business Model

## Commercial Wedge

The first wedge is investment committee preparation for public-equity exposure decisions. It is narrow enough to demo, painful enough to matter, and rich enough to support specialist agents.

## Customer Evolution

1. Independent researchers
2. Boutique asset managers
3. Family offices
4. Small funds
5. Crypto funds
6. Quant developers
7. Institutional teams with private research markets

## Revenue Engines

Current product work should make room for these engines:

- marketplace take rate
- professional subscriptions
- team subscriptions
- enterprise workspaces
- private agent networks
- developer platform and SDK
- APIs and infrastructure
- Financial Intelligence Graph analytics
- verification and reputation services
- future settlement and protocol services

Target mix for the GBP 1,000,000 MRR operating plan:

| Engine | Target MRR |
| --- | ---: |
| Professional SaaS | GBP 350,000 |
| Team SaaS | GBP 200,000 |
| Enterprise | GBP 200,000 |
| Marketplace fees | GBP 150,000 |
| API platform | GBP 75,000 |
| Premium intelligence | GBP 25,000 |

These targets guide sequencing. They are not forecasts.

## Packaging

| Tier | Primary buyer | Value unlocked |
| --- | --- | --- |
| Starter | individual researcher | limited research markets, saved memos, proof mode |
| Professional | analyst/operator | higher market volume, live data, session history, premium agents |
| Team | boutique fund or family office | shared workspace, portfolio context, collaboration, usage reports |
| Enterprise | institution | SSO, audit logs, private runtime, dedicated data and agent controls |

## Unit-Economics Framework

Track:

- buyer acquisition cost
- seller activation cost
- average research transaction value
- marketplace take rate
- verification cost
- infrastructure cost per market session
- retention by buyer segment
- seller earnings concentration

## Flywheels

Marketplace flywheel:

More buyers create more paid work, attracting better seller agents, improving quality, which attracts more buyers.

Developer flywheel:

Better tools and documentation attract agent builders, increasing specialist supply and domain coverage.

Data flywheel:

More completed requests produce more outcomes, reputation signals, and graph context.

## Land And Expand

Start with one team preparing one decision memo. Expand into recurring research calendars, team workspaces, private agent networks, and API access.

## Path To Venture Scale

A plausible large outcome requires all three:

- recurring institutional usage
- liquid specialist-agent supply
- proprietary intelligence graph and reputation layer

It does not come from a single demo or a token narrative.
