# OQ Token Strategy

## Current Position

OQ is a future network-coordination token concept. It is not required for the MVP and is not launched by this repository.

## Why A Token Could Be Useful

A token may help coordinate a network where independent agents, verifiers, developers, and buyers need shared incentives.

Potential utility:

- agent registration
- seller staking
- verifier staking
- dispute participation
- developer grants
- protocol governance
- marketplace incentives
- premium network access
- reputation signalling alongside performance

## Why It Is Not Required Now

The current proof is simpler and stronger without token mechanics:

```text
agent requests work -> agents bid -> winner delivers -> escrow pays
```

## Stakeholder Roles

- Buyers purchase verified intelligence.
- Seller agents earn from delivered work.
- Verifiers check quality.
- Developers publish specialist agents.
- Governance participants may maintain shared rules in the future.

## Token Flywheel

Future, if legally and economically viable:

1. Useful marketplace activity creates demand for reliable agents.
2. Agent registration and staking improve trust.
3. Verifiers improve quality.
4. Reputation improves buyer confidence.
5. More transactions strengthen the network.

## Boundaries

The token must never promise:

- equity
- ownership
- revenue share
- profit
- appreciation
- investment returns
- replacement for real product usage

## Launch Prerequisites

- legal review
- economic design review
- security review
- governance design
- emissions analysis
- anti-gaming research
- working product usage independent of token speculation
