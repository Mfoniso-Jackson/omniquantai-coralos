# Distribution Playbook

## Purpose

Distribution should compound credibility, developer adoption, institutional trust, network effects, proprietary data, marketplace liquidity, and long-term enterprise value.

OmniQuantAI is the Financial Intelligence Network: autonomous agents competing to create and exchange financial intelligence.

## Six Distribution Engines

### Engine 1: Build In Public

Every release should generate:

1. GitHub release note
2. changelog entry
3. short architecture note
4. demo clip or screenshot
5. LinkedIn post
6. X thread
7. newsletter or mailing-list update when available

Never ship silently.

### Engine 2: Developer Ecosystem

Goal: a developer should understand how to build a specialist financial agent in under one hour.

Assets:

- `docs/agent-builder-guide.md`
- API docs
- marketplace protocol docs
- sample seller personas
- future marketplace submission guide
- good first issues

### Engine 3: Financial Intelligence Content

Own topics:

- multi-agent finance
- Financial Intelligence Graphs
- agent reputation
- market mechanism design
- AI safety in finance
- programmable settlement

### Engine 4: Design Partners

Use `docs/design-partner-program.md` to recruit independent researchers, boutique funds, family offices, crypto funds, university finance labs, and quant researchers.

### Engine 5: Community

Community should produce agents, not just chat. Use `COMMUNITY_PLAYBOOK.md`.

### Engine 6: Enterprise

Enterprise motion starts with a narrow workflow: investment committee preparation for NVIDIA exposure decisions. Use `ENTERPRISE_SALES.md`.

## Release Distribution Checklist

For every meaningful release:

- update `CHANGELOG.md`
- run the relevant test gate
- capture one screenshot or terminal proof
- write a three-line release note
- identify one developer-facing lesson
- identify one institutional-trust improvement
- add one follow-up issue
