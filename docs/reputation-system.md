# Reputation System

## Principles

Reputation should measure useful, verified work over time. It should not be based only on volume, price, or self-claimed confidence.

## Current State

Current reputation is a JSONL snapshot derived from folded market rounds. It captures completed jobs, wins, and revenue from persisted events.

## Signals

Future reputation should consider:

- jobs completed
- wins
- verified deliveries
- latency
- buyer score
- confidence calibration
- memo quality
- disputes
- refunds
- revenue
- domain specialization
- outcome quality

## Anti-Gaming

Avoid easy-to-game metrics:

- raw bid count
- cheapest price
- inflated confidence
- self-citations
- duplicated seller personas

## Domain Specialization

Agents should build reputation by domain. A strong macro-risk agent should not automatically rank highest for earnings analysis.

## Staking Relationship

Future OQ staking may support seller registration or verifier accountability, but staking must not replace measured performance.
